课程：常见问题 (和解决方案)

编译问题

Microsoft Windows 系统常见错误信息

'javac' is not recognized as an internal or external command, operable program or batch file

如果您遇到这个错误，说明 Windows 找不到编译器 (javac)。

有一种办法可以告诉 Windows 去哪里查找 javac。假设您将 JDK 安装在 C:\jdk1.8.0，您需要输入下面的命令：

C:\jdk1.8.0\bin\javac HelloWorldApp.java

如果您这样做，那在每次使用 javac 和 java 命令时都需要冠以前缀 C:\jdk1.8.0\bin\。为避免这些额外的输入，查阅 JDK 8 安装指南中的 更新 PATH 变量 一节。

Class names, 'HelloWorldApp', are only accepted if annotation processing is explicitly requested

如果您遇到这个错误，说明在编译程序时忘记了加 .java 后缀。记住，命令是 javac HelloWorldApp.java 而不是 javac HelloWorldApp。

UNIX 系统常见错误信息

javac: Command not found

如果您遇到这个错误，说明 UNIX 找不到编译器 javac。

有一种办法可以告诉 UNIX 去哪里查找 javac。假设您将 JDK 安装在 /usr/local/jdk1.8.0。您需要输入下面的命令：

/usr/local/jdk1.8.0/javac HelloWorldApp.java

注意：如果您这样做，那在每次使用 javac 和 java 命令时都需要冠以前缀 /usr/local/jdk1.8.0。 为避免这些额外的输入，您可以将这个信息添加到 PATH 变量中。具体做法依赖于当前运行的是哪种 shell。

Class names, 'HelloWorldApp', are only accepted if annotation processing is explicitly requested

如果您遇到这个错误，寿命在编译时忘记了加 .java 后缀。记住，命令是 javac HelloWorldApp.java 而不是 javac HelloWorldApp。

语法错误 (所有平台)

如果程序中有输入错误，编译器会报 语法 错误。消息中通常会包含错误的类型，检测到错误那行的行号，这一行的代码，以及代码中出错的位置。 这里是一个在语句末尾省略分号 (;) 的错误：

Testing.java:8: error: ';' expected
            count++
                   ^
1 error

如果您看到了编译错误，表明程序并没有成功编译，当然也就不会生成 .class 文件。认真检查程序，修复错误，然后重新编译。

语义错误

除了验证程序在语法上的正确性，编译器也会检查其它基础的正确性。比如，编译器会警告您使用了未初始化的变量：

Testing.java:8: error: variable count might not have been initialized
            count++;
            ^
Testing.java:9: error: variable count might not have been initialized
        System.out.println("Input has " + count + " chars.");
                                          ^
2 errors

再次，您的程序没有成功编译，当然也就不会生成 .class 文件。修复错误，然后重新编译。

运行问题

Microsoft Windows 系统错误信息

Exception in thread "main" java.lang.NoClassDefFoundError: HelloWorldApp

如果您遇到这个错误，说明 java 找不到字节码文件 HelloWorldApp.class。

java 会尝试在当前目录中查找 .class 文件。如果您的 .class 文件在 C:\java， 您应该将当前目录修改为这个目录。要修改目录，输入下面的命令：

cd c:\java

提示应该变为了 C:\java。输入 dir 应该可以看到您的 .java 和 .class 文件。现在再次输入 java HelloWorldApp。

如果还有问题，可能需要修改 CLASSPATH 变量。使用下面的命令，查看是否有必要这么做。

set CLASSPATH=

现在再次输入 java HelloWorldApp。如果程序可以运行，说明已经修改了 CLASSPATH。要设置这个变量，查阅 JDK 8 安装指南中的 更新 PATH 变量 一节。这节也包含 CLASSPATH 变量的设置。

Could not find or load main class HelloWorldApp.class

初学者常犯的一个错误是尝试在 .class 文件上使用 java 启动器。比如，如果您使用 java HelloWorld.class 运行程序时会得到这个错误。记住，参数是 类的名称，而不是文件名。

Exception in thread "main" java.lang.NoSuchMethodError: main

JVM 要求您执行的类有个 main 方法，这是您的应用开始执行的地方。 详细剖析 "Hello World!" 应用 中详细讨论了 main方法。

UNIX 系统错误信息

Exception in thread "main" java.lang.NoClassDefFoundError: HelloWorldApp

如果您遇到这个错误，说明 java 找不到字节码文件 HelloWorldApp.class。

java 会尝试在当前目录中查找 .class 文件。如果您的字节码文件在 /home/jdoe/java，您应该将当前目录修改为这个目录。要修改目录，输入下面的命令：

cd /home/jdoe/java

在提示 pwd 时输入密码，您将看到 /home/jdoe/java。输入 ls 您将看到 .java 和 .class 文件。现在，再次输入 java HelloWorldApp。

如果还有问题，可能需要修改 CLASSPATH 变量。使用下面的命令，查看是否有必要这么做。

unset CLASSPATH

再次输入 java HelloWorldApp。如果程序可以运行，您必须修改 CLASSPATH 环境变量，和修改 PATH 变量一样。

Exception in thread "main" java.lang.NoClassDefFoundError: HelloWorldApp/class

初学者常犯的一个错误是尝试在 .class 文件上使用 java 启动器。比如，如果您使用 java HelloWorld.class 运行程序时会得到这个错误。记住，参数是 类的名称，而不是文件名。

Exception in thread "main" java.lang.NoSuchMethodError: main

JVM 要求您执行的类有个 main 方法，这是您的应用开始执行的地方。 详细剖析 "Hello World!" 应用 中详细讨论了 main方法。

Applet 或 Java Web Start 应用被阻止

如果您通过浏览器运行应用，并得到一条安全警告说应用被阻止了，清检查下面的内容：

  • 验证 JAR 清单文件中的属性，是否针对当前的运行环境正确设置。权限属性是必须的。在 NetBeans 项目中，点击 Files 选项卡，展开项目的文件夹，双击 manifest.mf 文件，打开清单文件。

  • 验证应用是否使用合法的证书签名，并且证书存放在 Signer CA keystore。

  • 如果您运行本地 applet，设置 web server 为用于测试。您也可以在 Java 控制面板的安全选项卡一栏中，将您的应用排除列表以外。
