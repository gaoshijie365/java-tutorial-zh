传递信息到方法或构造器

方法或构造器声明时，声明了其参数的数量和类型。比如，下面的方法根据贷款金额、税率、贷款时长 (分几期) 和贷款的未来价值，计算每个月份需要支付的房贷：

public double computePayment(
                  double loanAmt,
                  double rate,
                  double futureValue,
                  int numPeriods) {
    double interest = rate / 100.0;
    double partial1 = Math.pow((1 + interest), 
                    - numPeriods);
    double denominator = (1 - partial1) / interest;
    double answer = (-loanAmt / denominator)
                    - ((futureValue * partial1) / denominator);
    return answer;
}

这个方法有四个参数：贷款金额、税率、未来价值和期数。前三个参数是双精度浮点数，第四个参数是整数。参数用在方法体中，在运行时会取传入的参数值。

注意： 形参 指的是方法声明中的变量列表。实参 调用方法时传入的实际值。当调用方法时，实参必须与声明的形参在类型和顺序上匹配。

参数类型

方法或构造器的参数可以是任何类型的。这包括基本数据类型，比如 double、float 和 int 等，也可以是引用类型，比如对象和数组。

这里有个方法，其接收数组作为参数。在本例中，方法创建了一个 Polygon 对象，并使用 Point 对象数组进行初始化 (假设 Point 是代表 x、y 坐标的类)：

public Polygon polygonFrom(Point[] corners) {
    // method body goes here
}
注意： 如果您想将一个方法传入另一个方法，请使用 lambda 表达式 或 方法引用。

任意数量的参数

您可以使用一个叫做 varargs 的结构向方法传递任意数量的值。当您不知道要向方法传递多少特定类型的参数时，使用 varargs。 它是手动创建数组的简便方式写法 (上一个方法可以使用 varargs 而不是数组)。

为使用 varargs，您需要在最后一个参数的类型后面跟着省略符号 (三个点，...)，然后是空格和参数名。然后就可以使用任意数量的参数调用该方法。

public Polygon polygonFrom(Point... corners) {
    int numberOfSides = corners.length;
    double squareOfSide1, lengthOfSide1;
    squareOfSide1 = (corners[1].x - corners[0].x)
                     * (corners[1].x - corners[0].x) 
                     + (corners[1].y - corners[0].y)
                     * (corners[1].y - corners[0].y);
    lengthOfSide1 = Math.sqrt(squareOfSide1);

    // more method body code follows that creates and returns a 
    // polygon connecting the Points
}

正如您所见，在方法中，corners 是被当做数组使用的。可以使用数组或者一些列参数调用方法，在方法内部都会被当做数组对待。

您将在常用的打印方法中看到 varargs。比如，这个 printf 方法：

public PrintStream printf(String format, Object... args)

它可以打印任意数量的对象。它可以这么调用：

System.out.printf("%s: %d, %s%n", name, idnum, address);

或这样

System.out.printf("%s: %d, %s, %s, %s%n", name, idnum, address, phone, email);

或是使用不同数量的参数。

参数名

当您给方法或构造器声明参数时，您需要为这个参数提供一个名字。方法体中的这个名字指向传入的实参。

参数名在其作用域内必须是唯一的。它不能与相同方法或构造器的另一个参数名称相同，也不能与方法或构造器中的局部变量名称相同。

参数名称可以和类字段名称相同。在这种情况下，参数 遮蔽 了字段。遮蔽字段会使代码难以阅读，习惯上仅用在构造器和字段的设值方法中。 比如，下面的 Circle 类和其 setOrigin 方法：

public class Circle {
    private int x, y, radius;
    public void setOrigin(int x, int y) {
        ...
    }
}

Circle 类有三个字段：x、y 和 radius。setOrigin 方法有两个参数， 其名称和字段名称相同。每个方法参数都遮蔽了同名的字段。所以在方法体中使用简单名 x 或 y 指的是参数，而不是字段。 为了访问字段，您必须使用限定名，这将在本课程的后续小节 "使用 this 关键字。" 中讨论。

传递基本数据类型参数

基本数据类型的参数，比如 int 或 double，是 按值 传入方法的。这也就意味着参数值的任何改变仅在方法的作用域内存在。 当方法返回后，参数就消失了，且任何改变都会丢失。这里有个例子：

public class PassPrimitiveByValue {

    public static void main(String[] args) {
           
        int x = 3;
           
        // invoke passMethod() with 
        // x as argument
        passMethod(x);
           
        // print x to see if its 
        // value has changed
        System.out.println("After invoking passMethod, x = " + x);
           
    }
        
    // change parameter in passMethod()
    public static void passMethod(int p) {
        p = 10;
    }
}

当运行这个程序时，输出：

After invoking passMethod, x = 3

传递引用数据类型参数

引用数据类型参数，比如对象，也是 按值 传入方法的。这也就意味着当方法返回后，传入的引用依然引用之前引用的对象。 然而，对象的字段的值 可能 在方法中改变了，如果适当的访问级别的话。

比如，考虑某个类中的一个方法，其移动 Circle 对象：

public void moveCircle(Circle circle, int deltaX, int deltaY) {
    // code to move origin of circle to x+deltaX, y+deltaY
    circle.setX(circle.getX() + deltaX);
    circle.setY(circle.getY() + deltaY);
        
    // code to assign a new reference to circle
    circle = new Circle(0, 0);
}

使用这些参数调用方法：

moveCircle(myCircle, 23, 56)

在方法内部，circle 最初指向 myCircle。方法改变了 circle 引用的对象的 x 和 y 坐标 (也就是 myCircle 的 x 和 y 坐标)，分别加了 23 和 56。当方法退出时，这些改变将会保留下来。 然后 circle 被赋值为一个新 Circle (x = y = 0) 对象的引用。然而，这次重新赋值没有保留下来，因为引用是按值传入的，不能改变。 在方法内部，circle 指向的对象已经改变，但是，当方法返回，myCircle 仍然引用相同的 Circle 对象，就像方法调用前那样。
