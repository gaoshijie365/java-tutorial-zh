理解类成员

本节我们讨论 static 关键字的使用，它用来创建属于类的字段和方法。

类变量

当同从一个类蓝图创建多个对象时，它们每一个都拥有自己的一份 实例变量 的拷贝。在 Bicycle 类中，实例变量是 cadence、gear 和 speed。每一个 Bicycle 对象都有这些变量的属于自己的值，存储在不同的内存位置。

有时候，您想要创建对所有对象都通用的变量。这通过 static 修饰符完成。static 修饰的字段叫做 静态字段 或 类变量。 它们和类相关，而不是对象。类的每个实例共享一个类变量，它在内存中有个固定的位置。任何对象都可以修改类变量的值，但是不创建类实例也可以操作类变量。

比如，假设您想创建一些 Bicycle 对象并给它们赋一个从 1 开始的序列号。每个对象的 ID 都是不一样的，所以它是一个实例变量。 同时，您需要一个字段来跟踪创建了多少个 Bicycle 对象，以便于计算下一个实例的 ID。这样的字段是全局的，不喝任何对象关联。 因此您需要一个类变量，numberOfBicycles，如下：

public class Bicycle {
        
    private int cadence;
    private int gear;
    private int speed;
        
    // add an instance variable for the object ID
    private int id;
    
    // add a class variable for the
    // number of Bicycle objects instantiated
    private static int numberOfBicycles = 0;
        ...
}

类变量通过类名引用，如

Bicycle.numberOfBicycles

这使得很容易识别出来它们是类变量。

注意： 您也可以使用对象引用静态字段 
myBike.numberOfBicycles
但不鼓励这么做，因为它没有清晰地表明这是个类变量。

您可以使用 Bicycle 的构造器设置 id 实例变量的值，并增加类变量 numberOfBicycles 的值：

public class Bicycle {
        
    private int cadence;
    private int gear;
    private int speed;
    private int id;
    private static int numberOfBicycles = 0;
        
    public Bicycle(int startCadence, int startSpeed, int startGear){
        gear = startGear;
        cadence = startCadence;
        speed = startSpeed;

        // increment number of Bicycles
        // and assign ID number
        id = ++numberOfBicycles;
    }

    // new method to return the ID instance variable
    public int getID() {
        return id;
    }
        ...
}

类方法

和静态变量一样，Java 也支持静态方法。静态方法是 static 修饰符修饰的方法，应该使用类名调用，如

ClassName.methodName(args)
注意： 您也可以使用对象引用调用静态方法 
instanceName.methodName(args)
但是不鼓励这么做，因为它没有清晰地表明这是个类方法。

静态方法的常见用途是访问静态字段。比如，我们可以为 Bicycle 类添加一个静态方法来访问静态字段 numberOfBicycles：

public static int getNumberOfBicycles() {
    return numberOfBicycles;
}

并不是所有的实例变量、实例方法、类变量、类方法的组合都被允许：

  • 实例方法可以直接访问实例变量和实例方法。
  • 实例方法可以直接访问类变量和类方法。
  • 类方法可以直接访问类变量和类方法。
  • 类方法 不能 直接访问实例变量或实例方法—必须使用对象引用。同样，类方法不能使用 this 关键字，因为并没有供 this 引用的实例。

常量

static 修饰符，结合 final 修饰符，也用来定义常量。final 修饰符表明字段的值不能改变。

比如，下面的变量声明定义了一个常量，叫做 PI，它的值接近 pi (圆的周长与直径比) 的值：

static final double PI = 3.141592653589793;

以这种方式定义的常量不能被重新赋值，如果您的程序尝试对其进行重新赋值，将会导致编译时错误。 习惯上，常量名全部大写。如果名字由多个单词组成，单词之间用下划线 (_) 分割。

注意： 如果一个基本类型或字符串定义为常量，并且其值编译时已知，编译器使用其值替换代码中的常量名。 这叫做 编译时常量。如果常量的值在外部世界中发生了变化 (比如，法律规定 pi 的实际值应该是 3.975)，您需要重新编译使用这个常量的类以获取当前值。

Bicycle 类

做完本节中的所有修改之后，Bicycle 现在如下：

public class Bicycle {
        
    private int cadence;
    private int gear;
    private int speed;
        
    private int id;
    
    private static int numberOfBicycles = 0;

        
    public Bicycle(int startCadence,
                   int startSpeed,
                   int startGear) {
        gear = startGear;
        cadence = startCadence;
        speed = startSpeed;

        id = ++numberOfBicycles;
    }

    public int getID() {
        return id;
    }

    public static int getNumberOfBicycles() {
        return numberOfBicycles;
    }

    public int getCadence() {
        return cadence;
    }
        
    public void setCadence(int newValue) {
        cadence = newValue;
    }
        
    public int getGear(){
        return gear;
    }
        
    public void setGear(int newValue) {
        gear = newValue;
    }
        
    public int getSpeed() {
        return speed;
    }
        
    public void applyBrake(int decrement) {
        speed -= decrement;
    }
        
    public void speedUp(int increment) {
        speed += increment;
    }
}
