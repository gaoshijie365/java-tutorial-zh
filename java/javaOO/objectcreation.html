创建对象

如您所知，类提供了创建对象的蓝图，您使用类创建对象。下面的代码片段节选自 CreateObjectDemo， 其中每一条语句都创建一个对象并赋值给一个变量：

Point originOne = new Point(23, 94);
Rectangle rectOne = new Rectangle(originOne, 100, 200);
Rectangle rectTwo = new Rectangle(50, 100);

第一行创建 Point 类的一个对象。 第二和第三行分别创建 Rectangle 类的对象。

每条语句都包含三部分 (下面详细讨论)：

  1. 声明：粗体 代码都是变量声明，包含变量名和对象类型。
  2. 实例化：new 关键字是 Java 中用来创建对象的操作符。
  3. 初始化：new 后面跟着一个构造方法调用，它初始化新对象。

声明指向对象的变量

您已经学习了变量的声明：

type name;

这告诉编译器，您将使用 name 指向类型为 type 的数据。对于基本类型变量，会为变量保留合适大小的内存。

您也可以在单独一行声明一个引用变量。如：

Point originOne;

如果您像这样声明 originOne，它的值将是不确定的，直到创建了一个对象并赋值给它。仅仅声明一个引用变量，并没有创建一个对象。 因此，您需要使用 new 操作符 (下节讲述)。在您使用 originOne 之前，必须先将一个对象赋值给它。否则，编译器将报错。

实例化一个类

new 操作符通过为新对象分配内存，并返回这块内存的引用来实例化一个类。new 操作符也调用对象的构造器。

注意："实例化一个类" 和 "创建一个对象" 意思相同。 当您创建一个对象时，就是在创建类的一个 "实例"，因此也叫 "实例化" 一个类。

new 操作符后跟一个参数：构造器的调用。构造器的名称提供了实例化的类的名称。

new 操作符返回创建的对象的引用。这个引用通常会赋值给合适类型的变量，比如：

Point originOne = new Point(23, 94);

new 操作符返回的引用也可以不赋值给变量。它可以直接用在一个表达式中。比如：

int height = new Rectangle().height;

这条语句将在下一节讨论。

初始化一个对象

Point 类的代码如下：

public class Point {
    public int x = 0;
    public int y = 0;
    //constructor
    public Point(int a, int b) {
        x = a;
        y = b;
    }
}

这个类只包含一个构造方法。构造方法很好识别，因为它的名字与类名相同并且没有返回类型。Point 类中的构造器接收两个整型参数 (int a, int b)。 下面的语句中使用 23 和 94 作为这些参数的值：

Point originOne = new Point(23, 94);

这条语句的执行结果可以用下图说明：

这里是 Rectangle 类的代码，包含四个构造器：

public class Rectangle {
    public int width = 0;
    public int height = 0;
    public Point origin;

    // four constructors
    public Rectangle() {
        origin = new Point(0, 0);
    }
    public Rectangle(Point p) {
        origin = p;
    }
    public Rectangle(int w, int h) {
        origin = new Point(0, 0);
        width = w;
        height = h;
    }
    public Rectangle(Point p, int w, int h) {
        origin = p;
        width = w;
        height = h;
    }

    // a method for moving the rectangle
    public void move(int x, int y) {
        origin.x = x;
        origin.y = y;
    }

    // a method for computing the area of the rectangle
    public int getArea() {
        return width * height;
    }
}

每个构造器都允许您使用基本类型和引用类型为矩形的原点、宽和高提供初始值。如果一个类有多个构造器，它们必须具备不同的签名。 Java 编译器基于参数的数量和类型区分构造器。当 Java 编译器遇到下面的代码，它知道调用的是 Rectangle 类中的要求一个 Point 参数后跟两个整型参数的构造器：

 
Rectangle rectOne = new Rectangle(originOne, 100, 200);

它调用 Rectangle 类的一个构造方法，并使用 originOne 初始化了 origin。同时，这个构造器将 width 设置为 100，将 height 设置为 200。现在有两个引用指向同一个 Point 对象—一个对象可以有多个引用指向它，如下图所示：

下面一行代码调用 Rectangle 的要求两个整型参数的构造器，并为 width 和 height 提供了初始值。 如果您查看构造器中的代码，它创建了一个 <x 和 y 值都被初始化为 0 的 Point 对象：

Rectangle rectTwo = new Rectangle(50, 100);

下面语句中使用的 Rectangle 构造器不接收任何参数，所以它叫做 无参构造器：

Rectangle rect = new Rectangle();

所有的类都至少有一个构造器。如果一个类没有显式地声明任何构造器，Java 编译器会自动提供一个无参构造器，叫做 默认构造器。 这个默认构造器调用父类的无参构造器，如果类没有其它父类，则调用 Object 的构造器。如果父类没有构造器 (Object 有一个)， 编译器将报错。
