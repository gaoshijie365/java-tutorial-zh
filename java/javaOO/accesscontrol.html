控制对类成员的访问

访问级别修饰符决定其它类是否可以使用特定的字段或者调用特定的方法。有两级访问控制：

  • 在顶级—public，或 package-private (没有显式的修饰符)。
  • 在成员级—public、private、protected、或 package-private (没有显式的修饰符)。

一个类可以使用修饰符 public 声明，这种情况下，这个类对所有类都可见。如果一个类没有修饰符 (默认，也叫做 包级私有)，它仅在自己的包中可见 ( 包是相关类的命名分组 — 您将在稍后课程中学习)。

在成员级，您可以使用 public 修饰符或者没有修饰符 (包级私有)，就和顶级类一样具有相同的意思。 对于成员，这里有两个额外的访问修饰符：private 和 protected。 private 修饰符指定成员仅可以在自己类中访问。 protected 修饰符指定成员仅可以在自己的包中访问 (与 包级私有 一样)，此外，也可以被另一个包中的子类访问。

下面的表格每个修饰符允许访问的成员。

Access Levels
Modifier Class Package Subclass World
public Y Y Y Y
protected Y Y Y N
no modifier Y Y N N
private Y N N N

第一个数据列指出类本身是否可以访问对应访问级别的成员。正如您所见，类总是可以访问自己的成员。第二列指出与类相同的包中的类可以访问的成员。第三列指出其它包中的子类是否可以访问对应级别的成员。第四列指出是否所有类都可以访问成员。

访问级别以两种方式影响您。第一，当您使用其它来源的类时，比如 Java 平台的类，访问级别决定了您可以使用这些类中的哪些的成员。 第二，当您写一个类，您需要决定您自己类的成员变量和方法应该具有的访问级别。

访问级别选择技巧： 

如果其他程序员使用您的类，您想要确保不会因为误用而产生错误。访问级别可以帮你做到。

  • 对特定成员使用最严格的访问级别是有意义的。使用 private 除非您有更好的理由不用它。
  • 避免 public 字段，常量除外。(本教程的示例中使用公有字段。这有助于简明地说明一些点，但是在生产代码中不推荐这么做。) 公有字段引导您到一个特定实现，并且会限制更改代码时的灵活性。
