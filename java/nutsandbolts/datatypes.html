基本数据类型

Java 是一门静态类型的编程语言，这也就意味着变量必须先声明后使用。变量的声明包括变量的类型和名称：

int gear = 1;

这表明有一个叫做 "gear" 的字段，持有整数数据，并且初始值为 "1"。变量的数据类型决定了它可以包含的值，以及可以对其执行的操作。 除了 int，Java 还支持其它 7 种 基本数据类型。基本类型是语言预定义的，并且由保留字命名。基本类的值之间不共享状态。Java 支持的 8 种基本数据类型是：

  • byte: byte 数据类型是 8 位有符号的二进制补码表示的整数。最小值为 -128，最大值为 127 (包括)。 byte 数据类型用在大 数组 中可以节省内存。 也可以用来替代 int 使代码更加清晰；事实上可以把一个变量的取值范围当做是一种形式的文档。

  • short: short 数据类型是 16 位有符号的二进制补码表示的整数。最小值为 -32768，最大值为 32767 (包括)。 和 byte 一样：可以用在大数组用 short 来节省内存。

  • int: 默认情况下，int 数据类型是 32 位有符号的二进制补码表示的整数，最小值为 -231，最大值为 231-1。从 Java SE 8 开始，您可以使用 int 数据类型表示一个无符号的 32 位整数，其最小值为 0，最大值位 232-1。 使用 Integer 类将 int 数据类型当作无符号整数使用。查看 Number 类一节获取更多信息。 Integer 类中添加了静态方法，如 compareUnsigned，divideUnsigned 等以支持无符号整数的算术运算。

  • long: long 数据类型是 64 位二进制补码表示的整数。有符号的 long 的最小值为 -263，最大值为 263-1。 从 Java SE 8 开始，您可以使用 long 数据类型表示一个无符号的 64 位整数，其最小值为 0，最大值为 264-1。 当您需要的值范围超过 int 所能表示的值时，使用 long 数据类型。 Long 类包含了方法，如compareunsigned，divideUnsigned 等以支持无符号 long 类型的算术操作。

  • float: float 数据类型是单精度 32 位 IEEE 754 浮点数。其取值范围超出了本节的讨论范围，但在 Java 语言规范的 浮点类型、格式和值 一节对其进行了说明。在大数组中使用 float 类型 (代替 double) 也可以节省内存。 这个数据类型不应该用来保存精确的值，比如货币。要表示精确的值，您可以使用 java.math.BigDecimal 类。 数字和字符串 一节讲解了 BigDecimal 和 Java 平台提供的其它非常有用的类。

  • double: double 数据类型是双精度 64 位 IEEE 754 浮点数。它的取值范围也在 Java 语言规范的 浮点类型、格式和值 一节讨论。double 类型通常是表示小数的默认选择。和 float 一样，您不应该用它来表示精确的值，比如货币。

  • boolean: boolean 数据类型有两个可能的值：true 和 false 。这个数据类型用来做一些简单的标记，跟踪 true/false 条件。虽然这个数据类型只代表 1 位的信息，但其占用内存大小并没有精确地定义。

  • char: char 数据类型表示一个 16 位的 Unicode 字符。其最小值为 '\u0000' (或 0)，最大值为 '\uffff' (或 65535)。

除了上面列出的 8 种基本数据类型，Java 也提供了 java.lang.String 类来支持字符串。将字符串放入双引号内将自动创建一个 String 对象，比如，String s = "this is a string";， 字符串对象是不可变的，这也就意味着一旦创建，它们的值将不会改变。从技术上来讲 String 类不是基本数据类型，但是考虑到其特殊性，您可以将其认为是一种基本数据类型。 您将在 简单数据对象 一节学习 String 类。

默认值

在声明字段时也可以不赋值。声明时没有赋初值的变量将由编译期设置一个默认值。一般来说，默认值将是 0 或 null，这取决于其数据类型。 依赖默认值被视为一种不好的编程风格。

下标总结了上述数据类型的默认值。

数据类型 默认值 (字段)
byte 0
short 0
int 0
long 0L
float 0.0f
double 0.0d
char '\u0000'
String (或任何对象)   null
boolean false

局部变量稍有不同，编译器不会为未初始化的局部变量赋一个默认值。如果您不能在局部变量声明时赋初值，就一定要确保在使用之前对其进行赋值。访问未初始化的局部变量将会导致编译时错误。

字面值

您肯呢个已经注意到了，在初始化基本类型时没有使用 new 关键字。基本类型是语言内置的数据类型，它们不是由类创建的对象。一个 字面值 就是一个固定值的源码表示。 如下所示，可以给基本类型的变量赋一个字面值：

boolean result = true;
char capitalC = 'C';
byte b = 100;
short s = 10000;
int i = 100000;

整数字面值

整数字面值如果以字母 L 或 l 结尾，其类型为 long，否则其类型为 int。 建议您使用大写字母 L，因为小写字母 l 与数字 1 很难区分。

整数类型 byte、short、int 和 long 的值可以由 int 字面值创建。 long 类型的值超出了 int 可以表示的范围，可以由 long 字面值创建。整数字面值可以由下面数字系统表示：

  • 十进制：基数为 10，包含 0 到 9 的数字，这也是您每天都在用的数字系统。
  • 十六进制：基数为 16，包含 0 到 9 的数字和 A 到 F 的字母。
  • 二进制：基数为 2，包含数字 0 和 1 (从 Java SE 7 开始您可以创建二进制的字面值)。

一般来讲，十进制系统是您可能用到的唯一数字系统。然而，如果您需要使用别的数字系统，下面的例子展示了正确的语法。0x 前缀表示十六进制， 0b 前缀表示二进制：

// The number 26, in decimal
int decVal = 26;
//  The number 26, in hexadecimal
int hexVal = 0x1a;
// The number 26, in binary
int binVal = 0b11010;

浮点字面值

浮点字面值如果以字母 F 或 f 结尾，其类型为 float；否则其类型为 double，可选地也可以加后缀 D 或 d。

浮点类型 (float 和 double) 可以使用 E 或 e 表示 (科学计数法)，F 或 f (32 位 float 字面值) 和 D 或 d (64 位 double 字面值，这是默认值，习惯上会省略)。

double d1 = 123.4;
// same value as d1, but in scientific notation
double d2 = 1.234e2;
float f1  = 123.4f;

字符和字符串字面值

char 和 String 字面值可包含任意的 Unicode (UTF-16) 字符。如果您的编辑器和文件系统支持，那您可以直接在代码中使用这样的字符。 如果不支持，您可以使用 "Unicode 转义" 比如 '\u0108' (带音调符号的 C)，或 "S\u00ED Se\u00F1or" (Sí 西班牙语中的 Señor)。 char 类型字面值用 '单引号' 括起来， String 字面值用 "双引号" 括起来。Unicode 转义序列可以用在程序的任何地方 (比如字段名)，而不仅仅是用在 char 或 String 字面值中。

Java 也支持一些特殊的转义序列： \b (退格)，\t (制表符)， \n (换行)， \f (换页)， \r (回车)， \" (双引号)， \' (单引号)， and \\ (反斜杠)。

还有一个 null 字面值，可以用作任何引用类型的值。null 可以赋值给任何非基本类型的变量。程序中经常使用 null 作为对象不可用的标记。

最后，还有一种特殊的字面值叫 类字面值，形式是类型名称加上后缀 ".class"，比如 String.class。指向代表类型本身的对象 (Class 类型)。

在数字字面值中使用下划线

从 Java SE 7 开始，数字字面值的数字之间可以有任意数量的下划线 (_)。这项特性让您可以对数字字面值中的数字分组，以提升其可读性。

比如，您的代码中有一个数字，包含很多位数，您可以使用下划线将其按照三个一组进行分割，就像您会使用逗号、空格作为分割符一样。

下面的例子展示了下划线在数字字面值中的使用：

long creditCardNumber = 1234_5678_9012_3456L;
long socialSecurityNumber = 999_99_9999L;
float pi =  3.14_15F;
long hexBytes = 0xFF_EC_DE_5E;
long hexWords = 0xCAFE_BABE;
long maxLong = 0x7fff_ffff_ffff_ffffL;
byte nybbles = 0b0010_0101;
long bytes = 0b11010010_01101001_10010100_10010010;

您只能将下划线放在数字之间，您不能将下划线放在下面的位置：

  • 数字的开始或结尾
  • 浮点字面值中与小数点相邻的位置
  • F 或 L 后缀前面
  • 期望出现一串数字的位置

下面的例子说明了在数字字面值中，合法与不合法的下划线的使用：

// Invalid: cannot put underscores
// adjacent to a decimal point
float pi1 = 3_.1415F;
// Invalid: cannot put underscores 
// adjacent to a decimal point
float pi2 = 3._1415F;
// Invalid: cannot put underscores 
// prior to an L suffix
long socialSecurityNumber1 = 999_99_9999_L;

// OK (decimal literal)
int x1 = 5_2;
// Invalid: cannot put underscores
// At the end of a literal
int x2 = 52_;
// OK (decimal literal)
int x3 = 5_______2;

// Invalid: cannot put underscores
// in the 0x radix prefix
int x4 = 0_x52;
// Invalid: cannot put underscores
// at the beginning of a number
int x5 = 0x_52;
// OK (hexadecimal literal)
int x6 = 0x5_2; 
// Invalid: cannot put underscores
// at the end of a number
int x7 = 0x52_;
