变量

正如您在之前课程中所学，对象在其 字段 中存储状态。

int cadence = 0;
int speed = 0;
int gear = 1;

什么是对象？ 向您介绍了字段，但您可能还有一些问题，比如：字段的命名有什么规则和习惯？除了 int 其它数据类型是什么？声明字段的时候是否必须初始化？如果字段没有显式初始化，是否会为其赋一个默认值？ 我们将在本课程中揭晓这些问题的答案，但是在这么做之前，您必须意识到这里有一些技术上的区别。在 Java 中，同时使用了术语 “字段” 和 “变量”，初学者可能对它们感到疑惑，因为它们看起来指的是同一件事。

Java 中定义了下面几种变量：

  • 实例变量 (非静态字段) 从技术上来讲，对象在 “非静态字段” 中存储它们的状态，也就是说字段没有使用 static 关键字修饰。 非静态字段也叫做 实例变量，因为对于类的每个 实例 来说它们的值都是不同的；一个自行车的 currentSpeed 与另一个自行车的 currentSpeed 是无关的。
  • 类变量 (静态字段) 类变量 是带有 static 修饰符的字段；不管类被实例化多少次，都只存在一份变量。 对一个特定种类的自行车来说，可以将档位字段标记为 static，因为从概念上来说，它的所有实例都会有相同数量的档位。代码 static int numGears = 6; 可以创建这样一个字段。此外，也可以添加关键字 final，表明档位数永远不会改变。
  • 局部变量 对象在其字段中存储状态，类似地，方法经常在 局部变量 中存储其临时状态。声明局部变量的语法与声明字段类似 (比如，int count = 0;)。 没有给局部变量指派任何特殊的关键字；这是因为变量声明的位置在方法的开闭大括号之间，仅对声明它们的方法可见，在类的其余部分访问不到。
  • 参数 您已经见过参数的例子，在 Bicycle 类，在 "Hello World!" 应用的 main 方法中。 main 方法的签名是 public static void main(String[] args)。在这里，args 变量就是方法的参数。 重要的是参数总是被归为 "变量" 而不是 "字段"。这同样适用于其它接收参数的构造器 (比如构造器和异常处理器)，这部分内容将在后续教程中学习。

话虽如此，教程的剩余部分在讨论字段和变量时将使用下述通用指南。如果我们讨论 "一般的字段" (除了局部变量和参数)，我们简单的说 "字段"。 如果讨论适用于 "以上所有"，我们简单地称其为 "变量"。 如果上下文要求对其进行区分，我们将使用特定的术语 (静态字段、局部变量，等)。偶尔也会看到术语 "成员"。 一个类型的字段、方法和嵌套类型统称为其 成员。

命名

对于名称，每种编程语言都有自己的规则和习惯，Java 也一样。变量的命名规则和习惯可以总结如下：

  • 变量名大小写敏感。变量名可以是任何合法的标识符 — 不限长度的 Unicode 字母和数字序列，由字母、美元符 "$"、或下划线 "_" 开头。 习惯上，变量名以字母开头，而不是 "$" 或 "_"。此外，习惯上也从来不用美元符。您可能会在自动生成的名字中看到美元符，但是您不应该在自己的变量名中使用它。 相似的习惯也适用于下划线，技术上来说您的变量名可以以 "_" 开头，但不鼓励您这么做。不允许空白字符。
  • 后续的字符可以是字母、数字、美元符、或下划线。当为变量选择名字时，使用完整的单词而不是缩写。这么做可以使您的代码更易阅读和理解。大多数情况也也会使您的代码是自文档的，字段命名为 cadence、speed 和 gear，比缩写版，比如 s、c 和 g 更直白。 您选择的名字不能是 关键字或保留字。
  • 如果您选择的名字仅包含一个单词，那就全部小写。如果包含多个单词，后续每个单词的首字母大写。名称 gearRatio 和 currentGear 就是例子。 如果变量存储一个常量值，比如 static final int NUM_GEARS = 6，习惯上将所有字母大写，并使用下划线连接每个单词。习惯上，其它的地方不会用到下划线。
