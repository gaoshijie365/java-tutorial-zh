预定义注解类型

Java SE API 中预定义了很多注解类型。一些注解类型是供 Java 编译器使用的，还有一些应用到其它注解上。

Java 语言使用的注解类型

java.lang 中预定义的注解类型是 @Deprecated，@Override 和 @SuppressWarnings。

@Deprecated @Deprecated 注解表明被标注的元素是 废弃的，不应该再使用。当程序使用 @Deprecated 注解标注的方法、类或字段时，编译器会产生一个警告。当一个元素是废弃的， 应该使用 Javadoc @deprecated 标签文档化，如下面例子中所示。在 Javadoc 注释和注解中使用 at 符号 (@) 并不是巧合：它们在概念上是相关的。 同时，注意 Javadoc 标记以小写 d 打头，注解以大写 D 打头。

   // Javadoc comment follows
    /**
     * @deprecated
     * explanation of why it was deprecated
     */
    @Deprecated
    static void deprecatedMethod() { }
}

@Override @Override 注解通知编译器元素有意覆写父类中声明的元素。覆写方法将在 接口和继承 中讨论。

   // mark method as a superclass method
   // that has been overridden
   @Override 
   int overriddenMethod() { }

尽管覆写方法时并不要求使用这个注解，但它可以帮助避免错误。如果 @Override 标记的方法没有正确地覆写父类中的方法，编译器将产生一个错误。

@SuppressWarnings @SuppressWarnings 注解告诉编译器抑制特定的警告。在下面的例子中，使用了一个已废弃的方法，编译器通常会产生一个警告。在这个例子中，注解的使用导致警告被抑制了。

   // use a deprecated method and tell 
   // compiler not to generate a warning
   @SuppressWarnings("deprecation")
    void useDeprecatedMethod() {
        // deprecation warning
        // - suppressed
        objectOne.deprecatedMethod();
    }

每个编译器警告都属于一个种类。Java 语言规范列出了两个分类：deprecation 和 unchecked。unchecked 警告可能会在接口与引入 泛型 前的代码一起使用时产生。为了抑制多个类别的警告，使用下面的语法：

@SuppressWarnings({"unchecked", "deprecation"})

@SafeVarargs @SafeVarargs 注解，当应用到方法或构造器，断言代码没有在 varargs 参数上执行潜在不安全的操作。 当着个注解类型使用时，与 varargs 相关的未检查警告会被抑制。

@FunctionalInterface @FunctionalInterface 注解，在 Java SE 8 中引入，表明类型声明有意成为一个函数式接口，就像 Java 语言规范中定义的那样。

应用到其它注解上的注解

应用到其它注解上的注解叫做 元注解。java.lang.annotation 中定义了一些元注解类型。

@Retention @Retention 注解指定被标记的注解是如何存储的：

  • RetentionPolicy.SOURCE – 标记的注解仅在源码级存在，编译器会忽略之。
  • RetentionPolicy.CLASS – 标记的注解在编译时被编译器保留，但是会被 JVM 忽略。
  • RetentionPolicy.RUNTIME – 标记的注解被 JVM 保留，所以可以被运行时环境使用。

@Documented @Documented 注解表明每当指定的注解被使用时，这些元素应该被 Javadoc 工具文档化。(默认情况下，注解不会包含进 Javadoc。) 更多信息，查看 Javadoc 工具。

@Target @Target 注解标记另一个注解，来限制注解可以应用到哪种 Java 元素上。一个目标注解指定下面元素类型作为其值：

  • ElementType.ANNOTATION_TYPE 可以应用到注解类型。
  • ElementType.CONSTRUCTOR 可以应用到构造器。
  • ElementType.FIELD 可以应用到字段或属性。
  • ElementType.LOCAL_VARIABLE 可以应用到局部变量。
  • ElementType.METHOD 可以用用到方法级注解。
  • ElementType.PACKAGE 可以应用到包声明
  • ElementType.PARAMETER 可以应用到方法的参数。
  • ElementType.TYPE 可以应用到类的任何元素。

@Inherited @Inherited 注解表明注解类型可以从超类继承。(默认不可以继承。) 当用户查询注解类型并且类没有这种类型的注解时，那就会从类的超类查询这个注解类型。 这个注解只能用在类声明上。

@Repeatable @Repeatable 注解，Java SE 8 中引入，表明标记的注解可以在相同的声明或类型上使用多次。更多信息，查看 可重复注解。
