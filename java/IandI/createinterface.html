接口

在软件工程中，有许多场景需要不同小组的程序员遵循同一份契约，定义了它们的软件是如何交互的。每个小组都应只关注自己代码的编写，而不需要知道其他小组的代码。一般来讲，接口 就是这样的契约。

比如，想象在未来的，电脑控制的自动驾驶汽车通过城市道路运送乘客，而不需要人类操作。汽车制造商编写软件 (当然，是用 Java) 操作汽车—停、启、左转等等。另一个工业组，导航制造商， 使计算机系统可以接收 GPS (全球定位系统) 数据和通过无线传输交通状况，并使用这些信息来驾驶汽车。

汽车制造商必须公布一个工业标准接口，详细地定义调用哪些方法可以使汽车移动 (任何制造商的任何汽车)。然后，导航制造商可以调用接口中描述的方法来控制汽车。 工业组不需要知道其他组的软件是如何实现的。事实上，每个组高度拥有自己的软件，并具有在任何时间修改的权力，只要它继续遵循公开的接口。

Java 中的接口

在 Java 中 接口 是一个引用类型，与类相似，仅可以包含常量、方法签名、默认方法、静态方法和嵌套类型。只有默认方法和静态方法有方法体。 接口不能被实例化—它们只能被类 实现 或被其它接口 扩展。扩展在本课程后面讨论。

定义一个接口与创建一个类相似：

public interface OperateCar {

   // constant declarations, if any

   // method signatures
   
   // An enum with values RIGHT, LEFT
   int turn(Direction direction,
            double radius,
            double startSpeed,
            double endSpeed);
   int changeLanes(Direction direction,
                   double startSpeed,
                   double endSpeed);
   int signalTurn(Direction direction,
                  boolean signalOn);
   int getRadarFront(double distanceToCar,
                     double speedOfCar);
   int getRadarRear(double distanceToCar,
                    double speedOfCar);
         ......
   // more method signatures
}

注意方法签名没有大括号并且以分号结尾。

为了使用一个接口，您需要编写一个类来 实现 该接口。当一个可实例化的类实现了接口，它为接口中声明的每一个方法提供了方法体。比如，

public class OperateBMW760i implements OperateCar {

    // the OperateCar method signatures, with implementation --
    // for example:
    public int signalTurn(Direction direction, boolean signalOn) {
       // code to turn BMW's LEFT turn indicator lights on
       // code to turn BMW's LEFT turn indicator lights off
       // code to turn BMW's RIGHT turn indicator lights on
       // code to turn BMW's RIGHT turn indicator lights off
    }

    // other members, as needed -- for example, helper classes not 
    // visible to clients of the interface
}

在上面的自动驾驶汽车例子中，汽车制造商将实现接口。雪佛兰的实现将与丰田的不同，当然，两家制造商都遵循相同的接口。导航制造商是接口的客户， 将构建系统，使用汽车位置的 GPS 数据、数字街道地图和交通数据来驾驶汽车。导航系统将调用接口方法：turn、change lanes、brake、accelerate 等等。

作为 APIs 的接口

自动驾驶汽车例子，展示了接口作为一个工业标准 应用程序编程接口 (API)。APIs 在社交软件产品中也很常见。典型地，一家公司销售软件包，包含其他公司想要在自己的软件产品中使用的复杂方法。 一个例子可能是，一个数字图片处理方法包，售卖给制作终端用户图形程序的公司。图形处理公司编写自己的类实现一个接口，该接口对客户公开。图形公司然后调用接口中定义的图像处理方法。 尽管图形处理公司的 API 是公开的 (对他的客户)，其 API 的实现却是受保护的秘密—事实上，可能会在后续改进接口，只要继续实现客户依赖的原始接口。
