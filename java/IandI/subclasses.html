继承

在前面的课程中，您已经见过 继承 多次了。在 Java 中，类可以从其它类派生，从而 继承 这些类的字段和方法。

定义: 从另一个类派生的类叫做 子类 (也叫做 派生类，扩展的类 或 子类)。 子类派生自的类叫做 超类 (也叫做 基类 或 父类)。

除了 Object 没有超类外，每个类有且仅有一个直接超类 (单继承)。在没有任何其它显式超类的情况下，每个类隐式地是 Object 的子类。

类可以派生自那些派生自其它类的类，最后派生自顶层类 Object。这样的类可以说是继承链中所有类 (延伸到 Object) 的后代。

继承的概念虽然简单，但是很强大：当您想创建一个新类，但是已经有一个类包含你想要的一些代码，您可以从已有的类中派生出新类。这样做您就可以服用已有类中的字段和方法，而不用自己编写 (调试) 它们。

子类继承了超类的所有 成员 (字段、方法和嵌套类)。构造器不是成员，所以它们没有被子类继承，但是子类可以调用超类的构造器。

Java 平台类层次

Object 类，定义在 java.lang 包中，定义和实现了所有类的通用行为。在 Java 平台，很多类直接派生自 Object，其它类派生自这些类，等等，从而形成一个类层次。

Java 平台的所有类是 Object 的后代

在层次的顶端，Object 是所有类中最通用的。接近层次底部的类提供了更加特定的行为。

一个继承的例子

这儿是 Bicycle 类的一个可能实现 (Bicycle 类出现在 类和对象 课程中)： 

public class Bicycle {
        
    // the Bicycle class has three fields
    public int cadence;
    public int gear;
    public int speed;
        
    // the Bicycle class has one constructor
    public Bicycle(int startCadence, int startSpeed, int startGear) {
        gear = startGear;
        cadence = startCadence;
        speed = startSpeed;
    }
        
    // the Bicycle class has four methods
    public void setCadence(int newValue) {
        cadence = newValue;
    }
        
    public void setGear(int newValue) {
        gear = newValue;
    }
        
    public void applyBrake(int decrement) {
        speed -= decrement;
    }
        
    public void speedUp(int increment) {
        speed += increment;
    }
        
}

MountainBike 类是Bicycle 的子类：

public class MountainBike extends Bicycle {
        
    // the MountainBike subclass adds one field
    public int seatHeight;

    // the MountainBike subclass has one constructor
    public MountainBike(int startHeight,
                        int startCadence,
                        int startSpeed,
                        int startGear) {
        super(startCadence, startSpeed, startGear);
        seatHeight = startHeight;
    }   
        
    // the MountainBike subclass adds one method
    public void setHeight(int newValue) {
        seatHeight = newValue;
    }   
}

MountainBike 继承了 Bicycle 的所有字段和方法，并添加了 seatHeight 字段和其设值方法。 除了构造器，它就好像您从头开始写一个新的带有四个字段和五个方法的 MountainBike 类。然而，您并不需要做所有这些工作。 如果 Bicycle 类中的方法非常复杂并且以及你给花了大量的时间调试，这将非常有价值。

在子类中您可以做什么

不管子类在哪个包中，都继承了父类所有的 public 和 protected 成员。如果子类与其父类在同一个包中，也从父类继承了 package-private 的成员。 您可以原样使用继承的成员，替换它们，隐藏它们，或者使用新成员补充它们：

  • 继承的字段可以像任何其它字段那样直接使用。
  • 您可以在子类中声明一个与父类中字段同名的字段，这样就 隐藏 了父类中的字段 (不推荐)。
  • 您可以声明父类中没有的字段。
  • 可以直接使用继承的方法。
  • 您可以在子类中写一个与父类中签名相同的 实例 方法，这就 覆写 了父类中的那个方法。
  • 您可以在子类中写一个与父类中签名相同的 静态 方法，这就 隐藏 了父类中的那个方法。
  • 您可以在子类中声明一个父类中不存在的方法。
  • 您可以在子类中写一个调用超类构造器的构造器，不管是隐式地还是使用 super 关键字。

本课程中的后面小节将展开讲解这些主题。

超类中的私有成员

子类没有继承父类的 private 成员。然而，如果超类有公有或受保护的方法来访问其私有字段，那子类也可以使用这些方法。

嵌套类可以访问外围类的所有私有成员—包括字段和方法。因此，被子类继承的共有或受保护嵌套类间接地访问超类的所有私有成员。

转换对象

对象的类型是实例化它的类，如果我们写

public MountainBike myBike = new MountainBike();

myBike 的类型是 MountainBike。

MountainBike 是 Bicycle 和 Object 的后代。因此 MountainBike 是 Bicycle 同时也是 Object，它可以用在任何调用 Bicycle 或 Object 的地方。

反过来就不一定行：Bicycle 可能是 MountainBike，但也可能不是。类似地，一个 Object 可能是 Bicycle 或 MountainBike，也可能不是。

转换 展示了一种类型的对象代替另一种类型的对象的使用，只要对象被继承和实现允许。比如，如果我们写

Object obj = new MountainBike();

obj 既是 Object 也是 MountainBike (直到 obj 被赋值为另一个 不是 MountainBike 类型的对象)。这叫做 隐式转换。

另一方面，如果我们写

MountainBike myBike = obj;

我们将得到一个编译时错误，因为编译期不知道 obj 是一个 MountainBike。然而，我们可以通过 显式转换 告诉 编译器， 我们承诺给 obj 赋值的是一个 MountainBike：

MountainBike myBike = (MountainBike)obj;

这个转换会在运行时检查 obj 被赋值为一个 MountainBike，这样编译器就可以安全地假定 obj 是一个 MountainBike。如果 obj 不是一个 MountainBike，将会抛出一个运行时异常。

注意： 您可以使用 instanceof 操作符编写逻辑来测试特定对象的类型。这可以避免运行时的错误。比如： 
if (obj instanceof MountainBike) {
    MountainBike myBike = (MountainBike)obj;
}

instanceof 操作符检查 obj 指向一个 MountainBike，所以我们可以执行转换，并且不会有运行时异常抛出。
