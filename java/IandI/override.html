覆写和隐藏方法

实例方法

子类中的实例方法，其签名 (名称以及参数的数量和类型) 和返回类型与父类中的实例方法相同，就 覆写 了父类的方法。

子类覆写方法的能力，允许一个类从超类继承行为的同时对其进行适当的修改。覆写的方法与被覆写的方法，其名称、参数数量和类型、返回值类型都相同。 覆写的方法也可以返回被覆写方法返回类型的子类型。这个子类型叫做 协变返回类型。

覆写方法时，可以使用 @Override 注解告诉编译器你在覆写父类中的方法。如果编译器检测到父类中并不存在这样的方法，就会产生一个错误。 关于 @Override 的信息，参见 注解。

静态方法

如果子类定义了一个静态方法，其签名与父类中的静态方法相同，那子类中的方法就 隐藏 了超类中的那个方法。

隐藏一个静态方法和覆写一个实例方法的区别是：

  • 覆写实例方法调用的是子类中的版本。
  • 隐藏静态方法调用的版本取决于从超类还是子类调用。

考虑包含两个类的例子。第一个是 Animal，包含一个实例方法和一个静态方法：

public class Animal {
    public static void testClassMethod() {
        System.out.println("The static method in Animal");
    }
    public void testInstanceMethod() {
        System.out.println("The instance method in Animal");
    }
}

第二个类，是 Animal 的子类，叫做 Cat：

public class Cat extends Animal {
    public static void testClassMethod() {
        System.out.println("The static method in Cat");
    }
    public void testInstanceMethod() {
        System.out.println("The instance method in Cat");
    }

    public static void main(String[] args) {
        Cat myCat = new Cat();
        Animal myAnimal = myCat;
        Animal.testClassMethod();
        myAnimal.testInstanceMethod();
    }
}

Cat 类覆写了 Animal 类中的实例方法，并且隐藏了 Animal 类中的静态方法。 这个类中的 main 方法创建了一个 Cat 实例，并调用类的 testClassMethod() 方法和实例的 testInstanceMethod 方法。

程序的输出如下：

The static method in Animal
The instance method in Cat

调用的是超类中的隐藏静态方法，和子类中的覆写实例方法。

接口方法

接口中的 默认方法 和 抽象方法 就像实例方法一样被继承。 然而，当超类或超接口提供了多个签名相同的默认方法时，Java 编译器遵循继承规则来解决命名冲突。这些规则由以下两个原则导出：

  • 实例方法优先于接口默认方法。

    考虑下面的类和接口：

    public class Horse {
    public String identifyMyself() {
        return "I am a horse.";
    }
}
    public interface Flyer {
    default public String identifyMyself() {
        return "I am able to fly.";
    }
}
    public interface Mythical {
    default public String identifyMyself() {
        return "I am a mythical creature.";
    }
}
    public class Pegasus extends Horse implements Flyer, Mythical {
    public static void main(String... args) {
        Pegasus myApp = new Pegasus();
        System.out.println(myApp.identifyMyself());
    }
}

    Pegasus.identifyMyself 方法返回字符串 I am a horse.

  • 已被其他候选对象覆盖的方法将被忽略。当超类型共享一个共同的祖先时，可能会出现这种情况。

    考虑下面的接口和类：

    public interface Animal {
    default public String identifyMyself() {
        return "I am an animal.";
    }
}
    public interface EggLayer extends Animal {
    default public String identifyMyself() {
        return "I am able to lay eggs.";
    }
}
    public interface FireBreather extends Animal { }
    public class Dragon implements EggLayer, FireBreather {
    public static void main (String... args) {
        Dragon myApp = new Dragon();
        System.out.println(myApp.identifyMyself());
    }
}

    Dragon.identifyMyself 方法返回字符串 I am able to lay eggs.

如果两个或更多独立定义的默认方法产生了冲突，或者默认方法与抽象方法产生了冲突，Java 编译器就会报错。您必须显式地覆写超类型的方法。

考虑可以飞的自动驾驶汽车。您有两个接口 (OperateCar 和 FlyCar) 提供了相同方法的默认实现，(startEngine)：

public interface OperateCar {
    // ...
    default public int startEngine(EncryptedKey key) {
        // Implementation
    }
}
public interface FlyCar {
    // ...
    default public int startEngine(EncryptedKey key) {
        // Implementation
    }
}

同时实现 OperateCar 和 FlyCar 接口的类必须覆写 startEngine 方法。您可以使用 super 关键字调用任何接口中提供的默认实现。

public class FlyingCar implements OperateCar, FlyCar {
    // ...
    public int startEngine(EncryptedKey key) {
        FlyCar.super.startEngine(key);
        OperateCar.super.startEngine(key);
    }
}

super 之前的名字 (在本例中是 FlyCar 或 OperateCar) 必须指向定义或继承被调用方法的默认实现的直接超接口。 这种形式的方法调用不受限于包含相同签名默认方法的多个接口。您可以使用 super 关键字调用类和接口中的默认方法。

从类继承的实例方法可以覆写抽象接口方法。考虑下面的接口和类：

public interface Mammal {
    String identifyMyself();
}

public class Horse {
    public String identifyMyself() {
        return "I am a horse.";
    }
}

public class Mustang extends Horse implements Mammal {
    public static void main(String... args) {
        Mustang myApp = new Mustang();
        System.out.println(myApp.identifyMyself());
    }
}

Mustang.identifyMyself 方法返回字符串 I am a horse. 。Mustang 类从 Horse 类继承了方法 identifyMyself，它覆写了接口 Mammal 接口中的同名抽象方法。

注意：接口中的静态方法不会被继承。

修饰符

覆写的方法的访问范围只能比被覆写的方法相同或更大。比如，超类中一个受保护的实例方法，在子类中被覆写后只能是公有的，而不能是私有的。

如果您试图将超类中的实例方法，在子类中更改为静态方法，将会得到一个编译时错误，反过来也一样。

总结

下表 总结了当您定义与父类中签名相同的方法时会发生的事。

定义与父类中签名相同的方法
  父类实例方法 父类静态方法
子类实例方法 覆写 产生编译时错误
子类静态方法 产生编译时错误 隐藏
注意： 在子类中，您可以重载从父类继承的方法。这样的重载方法既没有隐藏也没有覆写父类中的实例方法—他们是新方法。
