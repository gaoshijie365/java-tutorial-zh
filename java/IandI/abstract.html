抽象方法和抽象类

抽象类 是声明为 abstract 的类—它可以包含也可以不包含抽象方法。抽象类不能被实例化，但它们可以被子类化。

抽象方法 是只有声明没有实现的方法 (没有大括号，其后跟分号)，就像：

abstract void moveTo(double deltaX, double deltaY);

如果一个类包含抽象方法，类本身 必须 声明为 abstract，如在：

public abstract class GraphicObject {
   // declare fields
   // declare nonabstract methods
   abstract void draw();
}

当抽象类被子类化时，子类通常提供父类中所有抽象方法的实现。然而，如果没有提供，子类也必须声明为 abstract。

注意： 接口 (查看 接口) 中未声明为默认或静态的方法，隐式地都是抽象的，所以接口中的方法没有使用 abstract 修饰符。(也可以使用，但没必要。)

抽象类与接口对比

抽象类很像接口。您不能实例化它们，它们可以包含带实现或不带实现的方法声明。然而，抽象类中可以声明非静态和非 final 的字段，定义公有，受保护和私有的具体方法。 接口中所有的字段都自动地是公有，静态和 final 的，声明或定义 (如默认方法) 的所有方法都是公有的。此外，您只能扩展一个类，不管它是不是抽象的，但是您可以实现多个接口。

应该使用哪个，抽象类或接口？

  • 考虑使用抽象类，如果下述陈述适用于您的情景：
    • 您想在几个紧密相关的类之间共享代码。
    • 您期望扩展自抽象类的类具有同样的字段或方法，或者要求其它访问修饰 (比如 protected 和 private) 符而不是 public。
    • 您想要声明非静态或非 final 的字段。这使您可以定义方法来访问和修改它们所属对象的状态。
  • 考虑使用接口，如果下述陈述适用于您的情景：
    • 您期望毫不相干的类实现您的接口。比如接口 Comparable 和 Cloneable 就被许多毫无关系的类实现了。
    • 您想要制定特定数据类型的行为，但是不关心谁实现了其行为。
    • 您想要利用类型的多继承优势。

JDK 中的一个抽象类的例子是 AbstractMap，它是集合框架的一部分。它的子类 (包括 HashMap，TreeMap 和 ConcurrentHashMap) 共享许多 AbstractMap 定义的方法 (包括 get，put， isEmpty，containsKey 和 containsValue)。

JDK 中的一个实现多个接口的例子是 HashMap，它实现了接口 Serializable，Cloneable 和 Map<K, V>。 通过阅读接口列表，您可以推断出 HashMap 实例 (无论是开发者或公司实现的类) 可以被克隆，可以序列化 (意思是可以转换成字节流， 查看 Serializable Objects)，以及一个映射具有的功能。 此外，Map<K, V> 接口已经使用许多默认方法，比如 merge 和 forEach 增强了，旧的已经实现这个接口的类就不需要再定义了。

注意，很多软件库同时使用了接口和抽象类，HashMap 类实现了几个接口，同时也扩展了抽象类 AbstractMap。

抽象类示例

在面向对象的绘图应用中，您可以绘制圆形、矩形、线、贝塞尔曲线和许多其它图形对象。这些对象都具有一些通用的状态 (比如，位置、方向、线的颜色、填充颜色) 和行为 (比如，移动、旋转、调整大小、绘制)，所有的图形对象都有一些相同的状态和行为 (比如：位置、填充颜色、移动)。其它的要求不同的实现 (比如，调整大小或绘制)。 所有 GraphicObject 必须可以绘制或调整大小，它们的区别仅在于如何实现。这是使用抽象类的绝佳场景。 您可以利用其相似性，并声明所有的图形对象都继承自同一个抽象父类 (例如，GraphicObject)，如 下图 所示。

Classes Rectangle, Line, Bezier, and Circle Inherit from GraphicObject

首先，声明一个抽象类 GraphicObject，提供所有子类共享的变量和方法，比如当前位置和 moveTo 方法。 GraphicObject 也声明了抽象方法，比如 draw 或 resize，这些方法需要被所有子类以不同的方式实现。 GraphicObject 类看起来类似：

abstract class GraphicObject {
    int x, y;
    ...
    void moveTo(int newX, int newY) {
        ...
    }
    abstract void draw();
    abstract void resize();
}

GraphicObject 的每个非抽象子类，比如 Circle 和 Rectangle，必须提供 draw 和 resize 方法的实现：

class Circle extends GraphicObject {
    void draw() {
        ...
    }
    void resize() {
        ...
    }
}
class Rectangle extends GraphicObject {
    void draw() {
        ...
    }
    void resize() {
        ...
    }
}

当抽象类实现接口时

在 接口 一节， 我们知道实现接口的类必须实现接口的 所有 方法。然而，我们也可以定义一个未实现接口所有方法的类，这样的类必须声明为 abstract。比如：

abstract class X implements Y {
  // implements all but one method of Y
}

class XX extends X {
  // implements the remaining method in Y
}

在本例中，X 类必须是 abstract，因为它并没有完全实现 Y 接口，但是类 XX 实现了 Y。

类成员

抽象类可能包含 static 字段和 static 方法。您可以通过类引用使用这些静态成员 (比如，AbstractClass.staticMethod ())。
