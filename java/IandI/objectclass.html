Object 作为超类

java.lang 包中的 Object 类，位于继承层次的最顶端。每个类都是 Object 类的直接或间接后代。你使用或编写的每个类都继承了 Object 的实例方法。 你不必使用这些方法，但是，如果你选择使用它们，就可能需要覆写它们并添加一些特定于你的类的代码。本节讨论的从 Object 中继承的方法是：

  • protected Object clone() throws CloneNotSupportedException
          创建并返回这个对象的一份拷贝。
  • public boolean equals(Object obj)
          表明其它对象是否与这个对象相等。
  • protected void finalize() throws Throwable
          当垃圾收集检测到没有引用指向对象时，由垃圾收集器调用。
  • public final Class getClass()
          返回对象的运行时类。
  • public int hashCode()
          返回对象的哈希值。
  • public String toString()
          返回对象的字符串表示。

Object 对象中的 notify、notifyAll 和 wait 方法，用于同步程序中独立运行线程的活动，将在课程稍后讨论。这是其中的五个方法：

  • public final void notify()
  • public final void notifyAll()
  • public final void wait()
  • public final void wait(long timeout)
  • public final void wait(long timeout, int nanos)
注意： 这些方法有微妙之处，尤其是 clone 方法。

clone() 方法

如果一个类，或其中一个超类，实现了 Cloneable 接口，您就可以使用 clone() 方法创建现存对象的副本。要创建副本，编写代码：

aCloneableObject.clone();

在 Object 中该方法的实现，会检查调用 clone() 方法的对象是否实现了 Cloneable 接口。如果没有实现，方法将抛出一个 CloneNotSupportedException 异常。异常处理将在后续课程中介绍。现在，您只需要知道 clone() 方法必须声明为

protected Object clone() throws CloneNotSupportedException

或者：

public Object clone() throws CloneNotSupportedException

如果您要编写一个 clone() 方法来覆写 Object 中的那个。

如果调用 clone() 方法的对象实现了 Cloneable 接口，Object 中的 clone() 方法实现，将创建与原始对象相同类型的对象，并将新对象的成员变量初始化为与原始对象对应的成员变量。

使您的类可克隆的最简单的方式是在您的类声明中添加 implements Cloneable。然后您的对象就可以调用 clone() 方法了。

对一些类来说，Object 中 clone() 方法的默认行为就够用了。然而，如果一个对象包含指向外部对象的引用，比如 ObjExternal，您可能需要覆写 clone() 方法以得到正确的行为。否则，ObjExternal 中的变化将对克隆对象可见。 这也就意味着原始对象和它的克隆对象并不是独立的—为了分离它们，您必须覆写 clone() 方法，使其克隆对象和 ObjExternal。 然后，原始对象引用 ObjExternal，克隆的对象引用 ObjExternal 的克隆，这样原始对象和克隆对象才真正独立开来了。

equals() 方法

equals() 方法比较两个对象的相等性，如果相等返回 true。Object 类中提供的 equals() 方法使用同一性操作符 (==) 来判定对象是否相等。对于基本数据类型，这将得到正确的结果。对于对象，则不会。Object 提供的 equals() 方法测试对象 引用 是否相等—也就是说，如果比较的对象确实是同一个对象。

为测试两个对象是相等的 (包含相同信息)，必须覆写 equals() 方法。这是一个例子，Book 类覆写了 equals() 方法：

public class Book {
    String ISBN;
    
    public String getISBN() { 
        return ISBN;
    }
    
    public boolean equals(Object obj) {
        if (obj instanceof Book)
            return ISBN.equals((Book)obj.getISBN()); 
        else
            return false;
    }
}

考虑这段测试两个 Book 类实例的相等性：

// Swing Tutorial, 2nd edition
Book firstBook  = new Book("0201914670");
Book secondBook = new Book("0201914670");
if (firstBook.equals(secondBook)) {
    System.out.println("objects are equal");
} else {
    System.out.println("objects are not equal");
}

即使 firstBook 和 secondBook 是两个不同对象，这段程序也显示 objects are euqal。 它们被认为是相等的，因为比较的对象包含相同的 ISBN 码。

如果同一性操作符不适合你的类，就应该覆写 equals() 方法。

注意： 如果你覆写了 equals() 方法，就要同时覆写 hashCode() 方法。

finalize() 方法

Object 类提供了 finalize() 回调方法，可能在对象变为垃圾时调用。Object 类的 finalize() 实现没有做任何事情—您可以覆写 finalize() 来做一些清理工作，比如释放资源。

finalize() 方法 可能 会被系统自动调用，但是它什么时候被调用，以及是否被调用都是不确定的。因此，不应该依赖这个方法来做清理。 比如，您在执行 I/O 操作后没有关闭文件描述符，并且您期望 finalize() 为你关闭它们，您可能耗尽文件描述符。您应该使用带资源的 try 语句自动关闭应用的资源。 查看 Java 平台，标准版 Hotspot 虚拟机垃圾收集指南 中的 try-with-resources 语句 和 终结器和弱、软、虚引用。

getClass() 方法

您不能覆写 getClass。

getClass() 方法返回一个 Class 对象，其包含可以用来获取类信息的方法，比如其名称 (getSimpleName())，其超类 (getSuperclass()) 和其实现的接口 (getInterfaces())。比如，下面的方法获取并显示对象的类名：

void printClassName(Object obj) {
    System.out.println("The object's" + " class is " +
        obj.getClass().getSimpleName());
}

java.lang 包中的 Class 类有很多方法 (超过 50 个)。比如，您可以测试一个类是注解 (isAnnotation())， 接口 (isInterface())，还是枚举 (isEnum())。您可以查看对象有哪些字段 (getFields()) 或有哪些方法 (getMethods())。等等。

hashCode() 方法

hashCode() 方法返回对象的哈希值，它是由哈希算法生成的一个整数值。

根据定义，如果两个对象相等，那它们的哈希值 必须 相等。如果您覆写了 equals() 方法，就改变了两个对象相等的条件，此时 Object 中实现的 hashCode() 就不再适用。因此，如果您覆写了 equals() 方法，也必须覆写 hashCode() 方法。

toString() 方法

您应该总是考虑覆写您类中的 toString() 方法。

Object 中的 toString() 方法返回代表对象的 String，这对调试非常有帮助。一个对象的 String 表示完全依赖于对象，这也就是为什么要在您的类中覆写 toString() 方法。

您可以使用 toString() 和 System.out.println() 打印一个对象的文本表示，比如一个 Book 类的实例：

System.out.println(firstBook.toString());

对于一个适当的覆写 toString() 方法，将打印一些有用的信息，比如：

ISBN: 0201914670; The Swing Tutorial; A Guide to Constructing GUIs, 2nd Edition
